使用自主搭建的服务器实现翻墙

科学优雅的VPS+SS翻墙教程

如果不打算购买服务器或者配置SS的话，可以尝试使用免费的赛风（这真的不是广告）

####点我有惊喜

氪金请继续

第一步：购买VPS服务器

点击下面的链接购买服务器将享受充50刀送25刀的优惠

####购买Vultr服务器

购买完成后来到控制台，选择Server并点击“+”号添加服务器

第二步：选择参数并部署服务器

如果你是在亚洲，请尽量选择新加坡、日本、或美国西海岸的服务器，不然数据传输延迟将很高
以下图为例
服务器地址：东京
系统：CentOS7 x64
硬件：25G SSD 单核 1G带宽 1000G的流量
其他的一律不选，点Deploy部署
费用是$5每月 （其实用来搭梯子有点浪费，但现在貌似没有$2.5的套餐了）

部署成功后等待服务器初始化，稍等片刻后，将在Server列表看到你添加的服务器
接着点击服务器，进入Server Information页面，查看用户名密码等详细信息

第三步：远程服务器添加翻墙脚本

在这一步开始前，你需要检查服务器是否可用，因为某些你懂的不可抗力因素，请按照以下步骤检查

1.检测Ping与丢包情况
详细的使用方法可以参考vultr官网的FAQ
2.检测IP端口是否开放
主要检查22端口，如果未开放的话就无法使用远程工具远程服务器，接下来也就无法操作了
3.如果上面有哪一步失败请返回步骤二重新选择服务器部署

检查完成后，请下载XShell连接上你的服务器

连接上服务器后，下载一键搭建ss脚本文件（因为我是用的flyzy的脚本搭建的ss，所以gitclone的链接如下）
在此也很感谢flyzy的站长写的脚本，确实简单好用，附上他的Blog地址https://www.flyzy2005.com/

1.在远程命令窗口的光标上复制命令

git clone https://github.com/flyzy2005/ss-fly

如果提示bash: git: command not found说明服务器上还没安装Git
执行yum -y install git安装

2.运行搭建SS的脚本代码

ss-fly/ss-fly.sh -i 替换成你的密码 443

为什么要选择443端口？因为现在很多自定义端口搭完服务器后会被秒ban，你也不想再来一次吧

安装完成后，你将会看到下图类似的场景

恭喜你，你的SS服务器已经搭建成功了！

第四步：下载shadowsocks（小飞机）配置客户端链接

SS的下载地址：下载

配置完成后右键小飞机，选择启用系统代理，代理模式为PAC
完成后就可以连上 “The True Internet”啦！！！恭喜！完结！撒花！

如果你是按照以上的步骤搭建的SS服务器，可以输入以下命令一键开启BBR加速

ss-fly/ss-fly.sh -bbr

注：BBR是 Google爸爸开源的一种拥塞控制算法，有兴趣可以自己搜索下

测试开启是否成功：

输入
sysctl net.ipv4.tcp_available_congestion_control
返回
net.ipv4.tcp_available_congestion_control = reno cubic bbr

输入
sysctl -n net.ipv4.tcp_congestion_control
返回
bbr

输入
ps -ef | grep bbr
返回类似
root 3627 3525 0 14:06 pts/0 00:00:00 grep --color=auto bbr
查看bbr是否已开启并加入进程中
(反正看返回的都带bbr就行啦)

现在BBR已成功开启，反正经我实验发现，BBR开了之后只是提速！该丢包还是丢包……
可能看服务器线路稳定情况吧……
