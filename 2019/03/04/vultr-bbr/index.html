简单了解下BBR

使用之前先简要介绍下BBR是什么东西

BBR是谷歌公司开发了一种优化TCP连接速度的算法，提高流量的发包效率和利用率，这样可以降低丢包，从而提高速度。并且有个很重要的特性，仅需单向优化，就可以大幅度提高性能。BBR的原理是：

这个BBR模块已经集成在4.9版本以后的linux内核中。
CentOS如果要使用，就需要更新到最新的内核。

它的谷歌项目地址是：https://github.com/google/bbr

服务器安装最新内核并开启BBR

示例基于Vultr+CentOS 7

1.更新系统
yum update -y

2.安装内核
rpm --import https://www.elrepo.org/RPM-GPG-KEY-elrepo.org rpm -Uvh http://www.elrepo.org/elrepo-release-7.0-2.el7.elrepo.noarch.rpm yum --enablerepo=elrepo-kernel install kernel-ml

3.查看安装内核并设置
awk -F\' '$1=="menuentry " {print i++ " : " $2}' /etc/grub2.cfg
一般情况下返回如下

4.启动设置 （0号位需要启动的内核，默认安装内核均是0）
就是修改grub启动文件，将新安装的内核设置为默认内核，然后重启机器
grub2-set-default 0

5.重启系统
reboot

重启后输入uname -a
查看内核是否为最新

6.vi编辑etc目录下的sysctl.conf文件
在末尾添加
net.core.default_qdisc = fq net.ipv4.tcp_congestion_control = bbr
两行，:wq保存并退出编辑

7.sysctl -p保存执行更改的参数

8.测试BBR开启是否成功：

输入
sysctl net.ipv4.tcp_available_congestion_control
返回
net.ipv4.tcp_available_congestion_control = reno cubic bbr

输入
sysctl -n net.ipv4.tcp_congestion_control
返回
bbr

输入
ps -ef | grep bbr
返回类似
root 3627 3525 0 14:06 pts/0 00:00:00 grep --color=auto bbr
查看bbr是否已开启并加入进程中
(反正看返回的都带bbr就行啦)

现在BBR已成功开启，反正经我实验发现，BBR开了之后只是提速！该丢包还是丢包……
可能看服务器线路稳定情况吧……
